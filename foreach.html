forEach：配列専用の「便利な道具」

forEach は、JavaScriptの文法ではなく、「配列（Array）が持っている機能（メソッド）」です。
「配列の中身を全部処理したい！」という、よくある作業を簡単にするために作られました。

1. 構文の仕組み

forEach は、配列の各要素に対して、指定した関数（コールバック関数）を一度ずつ実行します。

配列.forEach( (要素, インデックス) => {
    // 実行したい処理
});

仕組み

forEach メソッドは、配列の先頭から順番に要素を取り出し、関数に渡して実行します。

① コールバック関数 (Callback)
forEach に渡す関数のことです。この関数の中に、繰り返したい処理を記述します。
② 引数 (Arguments)
コールバック関数は、以下の引数を受け取ります。
1つ目: 現在処理している要素
2つ目: その要素のインデックス（番号）

覚えておきたい言葉

コールバック関数 (Callback Function)
「後で呼び出してもらう」ための関数のこと。forEach が裏側で「はい、次の要素！」と順番に呼び出してくれます。
引数 (Argument)
関数に渡すデータのこと。(animal) => ... の animal の部分です。好きな名前を付けられます。
アロー関数 (Arrow Function)
() => {} という矢印のような書き方のこと。短くて書きやすいので、forEach と一緒によく使われます。
const animals = ['Dog', 'Cat', 'Bird'];

animals.forEach( (animal, index) => {
    console.log(index + ":" + animal); 
    // 1つ目の引数(animal)に中身、2つ目(index)に番号が入ってきます
});

2. forEachのメリット

  • 読みやすい: 「配列をループしているんだな」と一目でわかります。
  • ミスが減る: i++ などを書かなくていいので、書き間違いが起きません。
  • 空っぽは無視: データが入っていない場所（疎な配列）は自動でスキップしてくれます。

3. 注意点（バスのルール）

便利な反面、オートマチックならではの制約もあります。

  • 🛑 途中下車できない (No break)
    break や continue は使えません。必ず最後まで走り切ります。
  • ⏳ 待ってくれない (No await)
    async/await を使っても、バスは次のバス停へ進んでしまいます。
    「処理が終わるのを待つ」ということができません（投げっぱなしになります）。
