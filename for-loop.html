for文：すべてを自分で操る「構文」

for文は、JavaScriptの「文法」として用意されている、最も基本的な繰り返しの命令です。
「どこから、どこまで、どうやって進むか」をすべて自分で細かく指定します。

1. 構文の仕組み（マニュアル操作）

for文は、3つのステップを自分で書くことで、ループの挙動を完全にコントロールします。
少し記述が長いですが、一つ一つの意味を理解すれば怖くありません。

for (初期化; 条件式; 更新式) {
    // 繰り返したい処理 (ブロック)
}

🚶 自分の足で歩くイメージ

for文は、ゴールに向かって一歩ずつ歩くことに似ています。

① 初期化 (Initialization)
「スタート地点に立つ」
最初に1回だけ実行されます。ここで「カウンター変数（歩数計）」を作ります。
例: let i = 0; （変数 i を作って 0 を入れる）
② 条件式 (Condition)
「まだゴールじゃないか確認する」
毎回、進む前にチェックします。これが true (正解) の間だけループが続きます。
例: i < 3; （i が 3 より小さいなら進む）
③ 更新式 (Update)
「一歩進む」
1回の処理が終わるたびに実行されます。カウンター変数を増やしたり減らしたりします。
例: i++ （i を 1 増やす）

覚えておきたい言葉

カウンター変数 (Counter Variable)
ループの回数を数えるための変数のこと。慣習として i (indexの頭文字) がよく使われます。
ブロック (Block)
{ ... } で囲まれた範囲のこと。「この中の処理を繰り返すよ」というグループ分けです。
インクリメント (Increment)
変数の値を 1 増やすこと。i++ は i = i + 1 と同じ意味です。
const animals = ['Dog', 'Cat', 'Bird'];

for (let i = 0; i < animals.length; i++) {
    console.log(animals[i]); // インデックスを使って要素を取り出す
}

2. for文にしかできないこと（メリット）

「面倒くさい」と思うかもしれませんが、すべて自分で決められるからこそ、自由自在です。
特に、以下の機能は forEach には存在しない、for文だけの特権です。

🛑 中断 (break)

「目的のものが見つかった！」という時に、ループを途中で強制終了できます。

⏭️ スキップ (continue)

「これは関係ないな」という時に、その回だけ飛ばして次へ進めます。

⏳ 待機 (await)

async/await を使って、通信などの処理が終わるのを「待ってから」次へ進めます。

🔄 柔軟な制御

後ろから数えたり (i--)、2つ飛ばしにしたり (i+=2)、自由自在です。
